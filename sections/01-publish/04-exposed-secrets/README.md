# 04-exposed-secrets - 静的ページの落とし穴（フロントに秘密を置かない）

「フロントだけの静的ページは安全側」ではあるものの、**フロントに置いたものは表示の有無に関係なく
全部ユーザーのブラウザに届く** という落とし穴を体感するための「やってはいけない例」デモです。

会員リストの画面ではメール・電話をマスクして表示していますが、元データは `public/main.js` に直書き
されており、**F12（開発者ツール）の Console / Sources / Network から丸見え** になります。

レクチャーの内容は [LECTURE.md](./LECTURE.md) を参照してください。

## 構成

```text
public/
  index.html   /   会員リスト（画面上はマスク表示）
  main.js          顧客データを直書き（＝F12で丸見えになる原因）
package.json       依存（wrangler）
```

## 起動方法

すべて **このフォルダ（`sections/01-publish/04-exposed-secrets/`）の中** で実行します。

### 準備

```bash
npm install
```

### ローカルで確認

```bash
npx wrangler pages dev ./public
```

表示された `http://localhost:8788`（ポートは変わることがあります）をブラウザで開き、**F12** を押して
コンソールに `customers` と打つと、マスク前の個人情報が丸ごと表示されます。

## 注意

- ここに入っている顧客データは **すべて架空** のサンプルです（`example.com` のメール等）。
- これは **アンチパターン（やってはいけない例）** の教材です。実運用でまねしないでください。
