# VPS vs SaaS

[前のページ](../02-onpremise-vs-cloud/LECTURE.md) で「クラウド（借りる）」を選びました。3 段階の第2段は、
その続きの問いです。**同じ"借りる"でも、サーバーを自分で管理するのか、それとも管理ごと任せるのか。**

クラウドと一口に言っても、借りられる "粒度" はいろいろあります。サーバーを 1 台まるごと借りて中身は
全部自分でやる形（VPS）もあれば、完成したサービスをただ使うだけの形（SaaS）もあります。その間にも
段階があります。ここでは、この **「管理の度合い」のグラデーション** を理解するのがゴールです。

今回のハンズオンで使った Cloudflare Workers / Pages が、このグラデーションのどこに位置するのかも
あわせて見ていきます。

## 説明

### グラデーションとして並べる

「VPS か SaaS か」は本当は二択ではなく、**管理をどこまで手放すか** の連続的なグラデーションです。
左に行くほど自由（でも全部自分）、右に行くほど手軽（でも決められた範囲で）になります。

```
自由・自己管理 ◀─────────────────────────────────▶ 手軽・おまかせ

  VPS / IaaS        PaaS・サーバーレス          SaaS
  （サーバーを        （アプリを置く台を          （完成品を
   1台借りる）         借りる）                   使うだけ）

  例: さくらのVPS     例: Cloudflare Workers     例: Gmail,
      AWS EC2            Pages, Vercel              Notion, freee
```

### それぞれの粒度

- **VPS / IaaS（サーバーを借りる）**
  仮想的なサーバーを 1 台まるごと借ります。OS の選択、ミドルウェア（Web サーバーや DB）の導入、
  セキュリティ更新まで **全部自分** で行います。自由度は最大ですが、運用の責任も最大です。
  さくらの VPS・ConoHa・AWS EC2 などが代表例。"常時動かしっぱなしのプログラム" も置けます。

- **PaaS・サーバーレス（アプリを置く台を借りる）**
  OS やサーバーの管理は事業者に任せ、利用者は **アプリのコードだけ** を載せます。OS 更新や
  セキュリティパッチを気にする必要がありません。今回使った **Cloudflare Workers / Pages**、そして
  **Vercel** がここに当たります。手軽な代わりに、実行時間の上限など "決められた範囲" の制約があります。

- **SaaS（完成品を使うだけ）**
  自分でアプリを作らず、できあがったサービスをそのまま使います。Gmail、Notion、freee などが代表例。
  「作る」のではなく「使う」側なので、開発の話とは少し層が違いますが、グラデーションの右端として
  押さえておくと位置関係がはっきりします。

### 責任共有という見方

この段は「**どこまでが自分の責任で、どこからが事業者の責任か**」という境界線の問題として見ると
すっきりします。右に行くほど、自分の責任範囲が狭くなります。

| 管理項目 | VPS / IaaS | PaaS・サーバーレス | SaaS |
|---|---|---|---|
| アプリのコード | 自分 | 自分 | 事業者（使うだけ） |
| ミドルウェア（Web サーバー等） | 自分 | 事業者 | 事業者 |
| OS・セキュリティ更新 | 自分 | 事業者 | 事業者 |
| ハードウェア | 事業者 | 事業者 | 事業者 |

自分の責任範囲が広いほど自由にできますが、その分やることが増えます。狭いほど楽ですが、決められた
枠の中で作ることになります。

:::warning
**「楽＝劣っている」ではありません。** 管理を任せると、自分でできることは確かに減ります。しかし
OS のセキュリティ更新を忘れて脆弱性を突かれる、といった事故のリスクも同時に手放せます。多くの
初学者にとっては、自由より「**事故りにくさ**」のほうが価値が大きいことが多いです。
:::

### 自由度と手軽さのトレードオフ

- **VPS が向く**：既存の Node / Express アプリをほぼそのまま動かしたい、常駐プロセスを持ちたい、
  OS やミドルウェアまで自分で握りたい。**サーバー管理を学ぶ気がある／それ自体が目的** な場合。
- **サーバーレス（Workers / Pages・Vercel）が向く**：とにかく早く・楽に公開したい、サーバー管理に
  時間を取られたくない、フロント＋軽い API＋DB という今回作ったような構成。**小さく始めたい** 場合。

### この段の結論

サーバー管理を学ぶこと自体が目的でなければ、**管理を任せる側（サーバーレス／マネージド）** から
始めるのが楽です。今回のハンズオンで Cloudflare Workers / Pages を使ってサーバーを一切管理せずに
公開できたのは、まさにこの "管理を任せる側" を選んでいたからです。

管理を任せる側に決めたら、最後の問いは「**では、どのベンダーに載せるのか**」です。Vercel・Cloudflare・
AWS・GCP・Azure を横並びで見るのが、第3段の話になります。
