# Web Analytics でアクセス解析

![Web Analytics でアクセス解析](./images/00-thumbnail.svg)

サイトを公開したら、次に気になるのは「どれくらい見られているのか」です。Cloudflare Web Analytics を使うと、プライバシーに配慮しながら、ページビューや訪問者数、参照元といったアクセス状況を手軽に把握できます。このレクチャーでは、これまでに公開した「ひとことボード」に Web Analytics を組み込み、アクセス解析を始める方法を学びます。

### Web Analytics とは

Cloudflare Web Analytics は、Cloudflare が無料で提供しているアクセス解析ツールです。「どれくらい見られているか」を **サッと把握する** ことに特化していて、ページビュー・訪問者数・参照元・国・人気ページといった基本的な指標を確認できます。

最大の特徴は、**訪問者を追跡するための Cookie を使わずに計測する** ことです。個人を追跡しないぶん、次のようなメリットがあります。

- **プライバシーに配慮できる** — 利用者を個人として追跡しない、訪問者に優しい設計です。
- **Cookie 同意バナーまわりが軽くなりやすい** — 追跡用の Cookie を使わないため、対応がシンプルになる場合があります。
- **導入が簡単で軽量** — スニペットを 1 行貼るだけで始められ、表示速度への影響もほとんどありません。

ダッシュボードでは、おもに次のような情報を確認できます。

| 項目 | 内容 |
| --- | --- |
| ページビュー | ページが表示された回数 |
| 訪問者数 | サイトを訪れたユーザーのおおよその数 |
| 参照元（リファラー） | どのサイトやサービスから来たか |
| 国 | 訪問者がどの国からアクセスしているか |
| 人気ページ | よく見られているページの一覧 |
| Core Web Vitals | LCP などの体感速度に関する指標 |

「アクセス数や人気ページをざっくり知りたい」ブログ・ドキュメント・個人サイトなどにちょうどよいツールです。一方で、ユーザーがどう動いて最終的にどれだけ成果につながったか、といった細かい分析には向きません（詳しくは後述の「[GA4 との違い](#ga4-との違い)」で触れます）。

### 導入する

Web Analytics の導入には、大きく **2 つの方式** があります。

- **スニペット方式** — 発行されたトークンを使い、計測用のスニペットを HTML に貼り付ける方法。どんなホスティング環境でも使えます。
- **自動方式（Automatic Setup）** — ドメインを Cloudflare のプロキシ配下（DNS 設定で **オレンジクラウド** が有効）で運用している場合に、ダッシュボードで有効化するだけで自動計測が始まる方法。

今回のように `*.pages.dev` のサブドメインで公開している場合はスニペット方式を使います。Cloudflare ダッシュボードの「Web Analytics」で対象サイトを追加するとトークンが発行されるので、次のスニペットをフロントの `public/index.html` の `</body>` 直前に貼り付けて再デプロイすれば計測が始まります。

```html
<script defer src="https://static.cloudflareinsights.com/beacon.min.js"
        data-cf-beacon='{"token": "発行されたトークン"}'></script>
```

![ブラウザに読み込まれたビーコンが計測データを Cloudflare のダッシュボードへ送る流れ](./images/01-beacon-flow.svg)

なお、サイトに Content Security Policy（CSP）を設定している場合は、`static.cloudflareinsights.com`（スクリプトの読み込み元）と `cloudflareinsights.com`（計測データの送信先）を許可しないとブロックされることがあります。

### ダッシュボードの見方

計測が始まると、Cloudflare ダッシュボードの「Web Analytics」から、期間を指定してページビューや訪問者数の推移をグラフで確認できます。人気ページや参照元、国別の内訳も一覧で表示されるので、「どこから来た人が、どのページをよく見ているか」を把握できます。

### Workers / Pages メトリクスとの違い

Cloudflare には、Web Analytics とは別に **Workers / Pages 自体のメトリクス** もあります。両者は見ている対象が異なります。

![Web Analytics は訪問者目線、Workers/Pages メトリクスは運用者目線という視点の対比](./images/02-analytics-vs-metrics.svg)

| | Web Analytics | Workers / Pages のメトリクス |
| --- | --- | --- |
| 見るもの | サイト訪問者の行動（ページビュー・参照元など） | サーバー側の挙動（リクエスト数・エラー率・実行時間など） |
| 視点 | 利用者目線のアクセス解析 | 運用者目線の稼働状況 |

「どれくらい見られているか」を知りたいときは Web Analytics、「アプリが正常に動いているか・エラーが出ていないか」を知りたいときは Workers / Pages のメトリクスを見る、と使い分けると分かりやすいです。

### GA4 との違い

アクセス解析ツールというと Google Analytics（GA4）が定番ですが、Web Analytics は「軽い版の GA4」ではありません。**そもそも見たいものが違うツール** です。

| | Cloudflare Web Analytics | Google Analytics (GA4) |
| --- | --- | --- |
| 得意なこと | アクセス状況をサッと把握する | ユーザー行動・導線・UX を深掘りして調査する |
| 計測の粒度 | 基本指標（PV・訪問者・参照元・国・人気ページ・Core Web Vitals） | イベント・コンバージョン・ユーザー行動フローまで細かく追える |
| Cookie / 同意 | Cookie レスで個人を追跡しない。同意バナーが軽くなりやすい | Cookie で個人・行動を追跡。同意バナーが必要になりやすい |
| 導入・運用 | スニペット 1 行で手軽。無料 | タグ設置・設定項目が多い。無料枠が中心 |

「ユーザーがどのページからどう動いて申し込みに至ったか」といった **UX や導線の調査をしたいなら、最初から GA4** が向いています。逆に「どれくらい見られているかをざっくり把握したい」だけなら、手軽で軽い Web Analytics で十分です。目的に合わせて選び、必要なら両方を併用しても構いません。
