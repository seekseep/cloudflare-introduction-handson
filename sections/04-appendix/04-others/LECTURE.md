# その他の便利な機能と料金の見極め方

![その他の便利な機能と料金の見極め方](./images/00-thumbnail.svg)

ここまでで、Cloudflare Pages・Workers・D1・R2・Turnstile・Web Analytics を使って、Web アプリを公開・運用するための基本的な機能を学んできました。

Cloudflare には、このほかにも便利な機能が数多く用意されています。どれも必要になったときに追加して使えるので、最初からすべて覚える必要はありません。

この章では、知っておくと役立つ便利な機能と、Cloudflare の料金の考え方について紹介します。まずは無料枠から始められるサービスが多いため、個人開発や学習用途でも安心して試すことができます。ます。

## 1. Workers KV — 軽量なキーバリュー保存

Workers KV は、「キー」と「値」の組み合わせでデータを保存する、とてもシンプルなデータ保存サービスです。世界中の Cloudflare のサーバーに分散して保存されるため、読み取りが速く、設定値や一時的なデータの保存に向いています。

D1 が表（テーブル）形式でデータを管理するデータベースなのに対し、Workers KV は「1 つのキーから 1 つの値を取り出す」用途に適しています。例えば、お知らせの表示内容、機能の ON/OFF 設定、アクセスカウンタなどを保存するときによく利用されます。

## 2. Cron Triggers — Worker の定期実行

Cron Triggers は、Worker を決まった時刻や一定の間隔で自動実行する機能です。サーバーを常に動かしておく必要がないため、定期的な処理を手軽に実行できます。

例えば、毎日古いデータを削除したり、1 日 1 回集計を行ったり、外部 API から最新の情報を取得したりするといった用途でよく利用されます。

## 3. Email Routing — 独自ドメインのメール受信・転送

Email Routing は、独自ドメイン宛に届いたメールを、普段使っているメールアドレスへ自動で転送できる機能です。例えば、`info@あなたのドメイン` 宛のメールを Gmail などで受け取れるようになります。

問い合わせ用のメールアドレスを簡単に作れるだけでなく、Workers と組み合わせることで、届いたメールをプログラムで自動処理することもできます。

## 4. Cloudflare Access（Zero Trust）— ページに認証をかける

Cloudflare Access は、Web ページや管理画面にログイン認証を追加できる機能です。Google アカウントなどで認証した人だけがアクセスできるように設定できます。

自分でログイン機能を開発しなくても利用できるため、管理画面や社内向けページ、関係者だけに公開したいページを手軽に保護できます。

## 5. Cache / Cache Rules — 配信を速く・安く

Cache は、一度取得したデータを Cloudflare のサーバーに一時的に保存し、次回以降はそのデータを返す仕組みです。毎回 Worker やサーバーで処理を行う必要がなくなるため、表示が速くなり、サーバーへの負荷も減らせます。

更新頻度の低い API や画像、CSS・JavaScript などの静的ファイルで特に効果があります。一方で、ユーザーごとに内容が異なるページや個人情報を含むデータは、キャッシュしないように設定することが大切です。

## 6. Workers AI — AI 推論を呼び出す

Workers AI は、Cloudflare 上で提供されている AI モデルを Worker から利用できる機能です。自分で AI サーバーを用意しなくても、文章の生成や要約、翻訳などの機能をアプリに組み込めます。

例えば、投稿内容の要約や翻訳、不適切な内容の判定、チャットボットなどを比較的簡単に実装できます。無料枠も用意されているため、まずは小さな機能から試してみるのがおすすめです。

## 7. Snippets / Redirect Rules — コードなしの軽い書き換え

Snippets や Redirect Rules は、プログラムを書かなくても、ダッシュボードの設定だけで Web サイトの動きを変更できる機能です。URL の転送や HTTP ヘッダーの追加など、簡単な処理を手軽に設定できます。

例えば、古い URL を新しい URL に転送したり、共通のセキュリティヘッダーを追加したりするときに便利です。簡単な設定であればコードを書く必要がないため、小さな変更を素早く反映できます。

## 無料枠

Cloudflare の多くのサービスには**無料枠**が用意されています。無料枠とは、「一定の利用量までは無料で利用できる」という仕組みです。学習や個人開発、小規模な Web サイトであれば、無料枠だけで十分利用できることも少なくありません。

アプリの利用者が増えたり、アクセス数やデータ量が増えたりすると、無料枠を超えて有料プランが必要になることがあります。そのため、最初は無料枠で始めて、サービスが成長してから必要に応じて有料プランへ移行する、という使い方が一般的です。

無料枠の内容や料金体系はサービスごとに異なり、変更されることもあります。最新の情報は、Cloudflare の公式ドキュメントを確認してください。

- [Workers の料金・無料枠（Workers・Pages・D1・KV など）](https://developers.cloudflare.com/workers/platform/pricing/)
- [Cloudflare の料金プラン一覧](https://www.cloudflare.com/plans/)

## 料金アラート

Cloudflare では、利用量や料金が一定の基準を超えたときに通知を受け取る**料金アラート**を設定できます。想定より利用量が増えた場合でも、メールなどで早めに気付くことができるため、予期しない課金を防ぎやすくなります。

特に有料プランを利用する場合や、アクセス数が増える可能性のあるサービスを公開する場合は、あらかじめ料金アラートを設定しておくと安心です。ただし、料金アラートは通知を行う機能であり、課金を停止したり利用を自動で制限したりするものではありません。

設定方法や利用条件については、Cloudflare の公式ドキュメントを確認してください。

- [Notifications](https://developers.cloudflare.com/notifications/)
- [Budget Alerts](https://developers.cloudflare.com/billing/manage/budget-alerts/)

## D1 の料金形態

D1 は、他のデータベースサービスとは少し異なる料金体系になっています。多くのサービスでは「CPU の使用時間」や「データの転送量」で課金されますが、D1 は**読み書きした行数（Rows Read / Rows Written）**と**保存しているデータ量**によって料金が決まります。:contentReference[oaicite:0]{index=0}

特に意識したいのが、**読み取った行数**です。例えば、100 万件のデータが入ったテーブルから 1 件だけ取得したい場合でも、検索方法が悪いと 100 万件すべてを調べてから目的のデータを見つけることがあります。この場合、取得した結果は 1 件でも、100 万行を読み取ったとして計算されます。:contentReference[oaicite:1]{index=1}

そのため D1 では、**必要なデータだけ取得する SQL を書くこと**が重要です。検索によく使う列にはインデックスを作成し、`WHERE` で条件を絞り込み、不要な `SELECT *` による全件取得を避けることで、処理速度だけでなく料金も抑えられます。インデックスを作成すると書き込みは少し増えますが、多くの場合は読み取り行数を大きく減らせるため、結果的にメリットの方が大きくなります。:contentReference[oaicite:2]{index=2}

無料枠や最新の料金体系については、公式ドキュメントを確認してください。

- [D1 Pricing](https://developers.cloudflare.com/d1/platform/pricing/)
